A. 减法操作

http://222.180.160.110:1024/contest/3887/problem/1

萌萌题。什么叫萌萌题呢，就是说他不给我大样例我还真以为这是道萌萌题。看到题面，我说，这不是根号找到最小质因子 $i$，然后直接输出 $n\div i$ 吗。

然后看到大样例吓傻了，我这和输出差了十万八千里啊。吓得重新看了一遍题。假设 $n=i\times \dfrac{n}{i}$，那么减了一次之后就变成了 $i\times (\dfrac{n}{i} - 1)$，此时 $(\dfrac{n}{i} - 1)$ 中就可能出现更小的质因子。那我岂不是要减一次分解一次？$10^{10}$ 直接 T 飞。说明会有更 NB 的方法。

这时关注到「最小」，我们知道最小的质数是 2，所以一旦 2 出现在 $n$ 的分解中，我们就可以一直减去它，因为没有比它更小的质数了。2 这个数呢又有一个特点，就是奇偶性。也就是说，因为我们分解 $n$ 的时候没有分出来 2，所以 $i$ 和 $\dfrac{n}{i}$ 都是奇数。我们按套路减去一个 $i$，那么就出现了 2 的倍数 $\dfrac{n}{i} - 1$，重新分解得 $n - i = 2\times \dfrac{i\times (\dfrac{n}{i} - 1)}2$。这时答案就是刚才减的那一次 1 再加上 $\dfrac{i\times (\dfrac{n}{i} - 1)}2$。

#define int long long
namespace XSC062 {
using namespace fastIO;
int n, x, y, z;
int main() {
    read(n);
    if (n % 2 == 0) {
        print(n / 2, '\n');
        return 0;
    }
    for (int i = 3; i * i <= n; ++i) {
        if (n % i == 0) {
            x = i, y = n / i - 1;
            z = x * y;
            print(z / 2 + 1, '\n');
            return 0;
        }
    }
    print(1);
    return 0;
}
} // namespace XSC062
#undef int

B. 匹配数

http://222.180.160.110:1024/contest/3887/problem/2

一点点鲜花

一开始我以为这是道 DP。

后来我发现 DP 假了于是自信差分约束。

最后其实差分约束是假的 DP 是真的而我也是真的，是真的 joker🤡

已加入 下饭操作合集。

说回来。为什么我会觉得 DP 假了呢？很简单，我是正着 DP 的，也就是说我先确定了前面的数位再向后 DP。可是这样后面的数位没办法决定选择哪个已有状态进行转移，因为我们没有办法仅凭上一位就得到哪个状态拥有最小的字典序。

但是题解很风轻云淡地给出了一个我一辈子想不出来的 solution：倒着 DP。我们只要保证每次选取最小的可行的下一位即可，这恰好符合字典序的定义。

记录前驱（or 后继？）后输出即可。

namespace XSC062 {
using namespace fastIO;
const int maxm = 15;
const int maxn = 2e3 + 5;
int n;
char s[maxn];
int f[maxn][maxm];
void output(int i, int j) {
    print(j);
    if (i != n)
        output(i + 1, f[i][j]);
    return;
}
int main() {
    scanf("%s", s + 1);
    n = strlen(s + 1) + 1;
    memset(f, -1, sizeof (f));
    for (int i = 0; i <= 9; ++i)
        f[n][i] = 1029;
    for (int i = n - 1; i; --i) {
        for (int j = 0; j <= 9; ++j) {
            if (s[i] == '>') {
                for (int k = j - 1; k >= 0; --k) {
                    if (~f[i + 1][k])
                        f[i][j] = k;
                }
            }
            else if (s[i] == '<') {
                for (int k = 9; k > j; --k) {
                    if (~f[i + 1][k])
                        f[i][j] = k;
                }
            }
            else if (s[i] == '=') {
                if (~f[i + 1][j])
                    f[i][j] = j;
            }
            else {
                for (int k = 9; ~k; --k) {
                    if (k == j)
                        continue;
                    if (~f[i + 1][k])
                        f[i][j] = k;
                }
            }
        }
    }
    for (int i = 1; i <= 9; ++i) {
        if (~f[1][i]) {
            output(1, i);
            break;
        }
    }
    return 0;
}
} // namespace XSC062

C. 染色

http://222.180.160.110:1024/contest/3887/problem/3

第二束鲜花

谁最讨厌数学题？我不讨厌数学题！谁的数学最拉跨？我的数学不拉跨！谁的嘴巴最梆硬？反正肯定不是我！🦆🦆🦆

这题解写得是生怕我看懂了 /fn 我摊牌了，我的数学最拉跨！

想了大半天复杂方法，没想到最后被逆向思维薄纱了。（智力：这难道不是很一眼吗？）（语气神似某讨打 C）

已加入 下饭操作合集。

不妨将一对相邻同色格子称为同色对，将一串连续的相同颜色称作一段。

那么最极端的情况下会有 $N$ 段（即不存在同色对）。

构造一段长度为 $L$ 的 $L$ 段序列（即不存在同色对）。此时将任意一段长度增加 1，则出现一对同色对。接下来，不论是选取长度为 2 的那一段，还是选取剩下的长度为 1 的段，将其长度增加 1，都会且仅会增加 1 对同色对。

以此类推，进行 $K$ 次「选取一段并将其长度增加 1」的操作，可得到刚好 $K$ 个同色对，此时序列长度为 $L + K$，而 段数没有变化，仍是一开始的 $L$。

所以反过来，如果在长度为 $N$ 的序列中存在 $K$ 个同色对，那么段数为 $N-K$。很难想象题解用「所以」两个字直接略过了上述推导过程

由此，我们只要考虑分别将序列任意分为非空的 $N-K\sim N$ 段，就可以解决问题，隔板法可得将 $N$ 分为 $T$ 段的方案数为 $C_{N-1}^{T-1}$。在此基础上考虑染色，由乘法原理，将 $T$ 段染色的方案为 $(m - 1)^{T - 1}\times M$。

取 $T=N-K\sim N$ 并求和问题即解决。

#define int long long
namespace XSC062 {
using namespace fastIO;
const int maxn = 2e5 + 15;
const int mod = 998244353;
int fac[maxn];
int n, m, k, res;
inline int qkp(int x, int y) {
    int res = 1;
    while (y) {
        if (y & 1)
            (res *= x) %= mod;
        (x *= x) %= mod;
        y >>= 1;
    }
    return res;
}
inline int inv(int x) {
    return qkp(x, mod - 2);
}
inline int A(int n, int m) {
    return (fac[n] * inv(fac[n - m])) % mod;
}
inline int C(int n, int m) {
    return (A(n, m) * inv(A(m, m))) % mod;
}
int main() {
    read(n), read(m), read(k);
    fac[0] = 1;
    for (int i = 1; i <= n + 5; ++i)
        fac[i] = (fac[i - 1] * i) % mod;
    for (int i = n - k; i <= n; ++i) {
        (res += ((C(n - 1, i - 1) *
                qkp(m - 1, i - 1)) % mod
                * m) % mod) %= mod;
    }
    print(res, '\n');
    return 0;
}
} // namespace XSC062
#undef int

D. 校庆纪念品

http://222.180.160.110:1024/contest/3887/problem/4

对于 50 分……

对于 50 分，不难想到运用小学数学知识，判断左端点和右端点分别包含多少个 $d$ 的倍数，若包含个数不同，则说明区间内存在 $d$ 的倍数（由前缀和思想，其实应判定的是 l - 1 而非 l）。

后来题解告诉我这个思想叫整除分块，还整得挺高大上的。

同样是用小学数学知识，我们知道当区间长度 $\ge d$ 时，必定会存在一个 $d$ 的倍数。当区间长度 $< d$ 时，$d$ 的倍数的存在性未知，但如果存在，就只有一个。

所以我们只用考虑长度 $< d$ 的区间。这里来了一个很妙的处理，我们直接找一个支持区间修改的数据结构，将该区间整体 $+1$。在遍历完 $N$ 个区间后，寻找 $M$ 内 $d$ 的倍数进行单点查询，结果之和就是答案。

由于 $d$ 的倍数在长度 $<d$ 区间内的唯一性，任意一个区间最多只会被统计一次。

但每次询问清空数据结构，重新统计 $N$ 个区间是很亏的，这里就有一个显而易见的点：$d$ 是不断增大的，我们将区间按长度排序，每次只用向数据结构内添加新的比 $d$ 短的区间即可。

每个区间最多只会被添加一次，故对于区间修改，时间复杂度 $\mathcal O(N \log M)$。

对于后面，查询 $M$ 内 $d$ 的倍数，就是我们熟知的调和级数了。调和级数自身的复杂度为 $\mathcal O(\ln M)$，再加之外层的大循环和数据结构单点查询，复杂度近似 $\mathcal O(N\log^2 M)$。于是乎最终时间复杂度 $\mathcal O(N\log^2 M)$。

namespace XSC062 {
#define lt (p << 1)
#define rt (lt | 1)
using namespace fastIO;
const int maxn = 3e5 + 5;
const int maxm = 1e5 + 5;
struct _ {
    int l, r;
    bool operator< (const _ &q) const {
        return r - l < q.r - q.l;
    }
};
int n, m;
_ a[maxn];
struct { int l, r, u, d; } t[maxm << 2];
void bld(int p, int l, int r) {
    t[p].l = l, t[p].r = r;
    if (l == r)
        return;
    int mid = (l + r) >> 1;
    bld(lt, l, mid);
    bld(rt, mid + 1, r);
    return;
}
inline void pushup(int p) {
    t[p].u = t[lt].u + t[rt].u;
    return;
}
inline void pushdown(int p) {
    t[lt].d += t[p].d;
    t[lt].u += (t[lt].r -
            t[lt].l + 1) * t[p].d;
    t[rt].d += t[p].d;
    t[rt].u += (t[rt].r -
            t[rt].l + 1) * t[p].d;
    t[p].d = 0;
    return;
}
void upd(int p, int l, int r) {
    if (l <= t[p].l && t[p].r <= r) {
        ++t[p].d;
        t[p].u += t[p].r - t[p].l + 1;
        return;
    }
    pushdown(p);
    int mid = (t[p].l + t[p].r) >> 1;
    if (l <= mid)
        upd(lt, l, r);
    if (r > mid)
        upd(rt, l, r);
    pushup(p);
    return;
}
int qry(int p, int x) {
    if (t[p].u == 0)
        return 0;
    if (t[p].l == t[p].r)
        return t[p].u;
    pushdown(p);
    int mid = (t[p].l + t[p].r) >> 1;
    if (x <= mid)
        return qry(lt, x);
    return qry(rt, x);
}
inline int getl(int i) {
    return a[i].r - a[i].l + 1;
}
int main() {
    read(n), read(m);
    for (int i = 1; i <= n; ++i)
        read(a[i].l), read(a[i].r);
    std::sort(a + 1, a + n + 1);
    bld(1, 1, m);
    for (int i = 1, j = 1; i <= m; ++i) {
        int res = 0;
        for (; j <= n && getl(j) < i; ++j)
            upd(1, a[j].l, a[j].r);
        res += n - j + 1;
        for (int k = i; k <= m; k += i)
            res += qry(1, k);
        print(res, '\n');
    }
    return 0;
}
} // namespace XSC062
