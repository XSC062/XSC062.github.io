总结 // 分块

两年前学习了分块，两年内没有任何一场模拟赛用到了这个知识点。于是我得出了结论：分块是毫无作用的。今年 lhy 又讲了一次，于是决定以博客记之。

说起来分块本来是一种思想，菌神等一众魔怔人却一致咬定它是一种 DS。真是奇怪啊。

首先不妨解答我一开始的问题：

为什么是分块？

不妨考虑一个数列的维护。

给出一个长为 $n$ 的数列，以及 $m$ 个操作，操作涉及区间加法，单点查值。

不难想到使用线段树或树状数组等数据结构。

那么考虑另一个问题。

给出一个长为 $n$ 的数列，以及 $m$ 个操作，操作涉及询问区间的最小众数。

此问题无法使用线段树或树状数组解决。原因：区间众数的取值 不满足可加性。

但是分块却可以解决该问题，因为分块从来 不需要可加性。它可以通过枚举每一个零散元素暴力合并状态。线段树为什么不能枚举左右子树呢？因为通常情况下的分块的零散元素最多只有二倍块长（块长通常情况下为 $\sqrt n$）个，而第一层的线段树左右子树的总大小为 $n$。所以线段树不行而分块可以。

思想介绍

分块的名字很直接，将需要维护的项目划分成块。每一个元素都一定属于且仅属于一个块。块中的元素都是连续的。形象地说，就是在项目中砍了几刀。

inline void Init(int n) {
    siz = sqrt(n);
    k = (n + siz - 1) / siz;
    for (int i = 1; i <= k; ++i) {
        t[i].l = t[i - 1].r + 1;
        t[i].r = t[i].l + siz - 1;
    }
    t[k].r = n;
    for (int i = 1; i <= n; ++i)
        f[i] = (i + siz - 1) / siz;
    return;
}
