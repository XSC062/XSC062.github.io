浅谈二分图

二分图简介

今天，我们来简单侃一侃二粉兔二分图。

二分图？和二分有什么关系嘛？

恭喜你，猜对了，二分图和二分毫无关系。（你猜我是不是觉得自己很幽默？

二分图是一种特殊的图论模型。

可以理解为这样一种图：能够将其中的所有点分成两组，其中每一组之间的所有点两两之间没有连边。

比如这样一个图：

我们把它分成 $1,3,5$ 和 $2,4,6$ 两个组，这样就可以让每个组内的点两两之间没有连边。

如图。

所以这就是一个二分图啦！

二分图的性质

二分图有一个有趣但对做题毫无帮助的性质，我们来水字数感性证明一下：

  • 若图 $G$ 中存在奇环，则 $G$ 不是二分图；反之，若 $G$ 中所有回路均具有偶数长度，则 $G$ 是一个二分图。

    什么是奇环？

    在图上，我们把有奇数个点的封闭图形称为奇环。（例如三角形，五边形等）

    贪心地想，如果 $G$ 中有两个联通的点，其中一个分到了一个组，另外一个点分到另一个组就行了。（事实上你也没有别的办法对吧

    那如果有一个环，从某一个点开始分组，如果这个环由偶数条边构成：

    那如果是奇数条边呢？

    剩下一个 $5$ 无家可归：它既不能到 $1$ 所在的那个组去，也不能到 $4$ 所在的那个组去。

    也就是说，如果一个环有偶数个点，要分到 $X,Y$ 两个组去，就可以分成 $X,Y,X,Y,\cdots,X,Y$ 的形式。（也可以是 $Y,X,Y,X,\cdots,Y,X$）

    反之，若有奇数个点，不管是 $X,Y,X,Y,\cdots,X$ 还是 $Y,X,Y,X,\cdots,Y$，起始点和终点这的点都会被分到同一组；而这显然是不符合定义的。

    至此，第一个问题就解决了；由于第二个有脑子就行，作者也不想在这些对做题没有太大帮助的地方浪费篇幅，所以我们就不多说了~~~（其实是你懒得写吧喂！~~

二分图的判定

上面那个定理其实只对人工判定二分图有帮助，电脑可不能一眼看出一个图有没有奇环。（辣鸡

所以我们只能深搜，或是广搜来模拟上面那个把“两个有边相连的点分到两个不同的组”的过程，如果模拟时发现分组有冲突，则这个图不是二分图。

所以有 DFS 版本：

int grp[maxn];		//Group
vector<int>g[maxn];	//Graph
bool dfs(int x){
    for(int i=0;i<g[x].size();++i){
        int e=g[x][i];
        if(grp[e]){				//若 e 分过组
            if(grp[e]==grp[x])	//x 和 e 有冲突
                return 0;		//返回非二分图
            continue;			//否则，e 已经访问过了，不用重复访问
        }
        grp[e]=3-grp[x];		//分成 1,2 两组
        if(!dfs(e))				//有一个部分不满足则整个图非二分图
            return 0;
    }
    return 1;
}
int main(){
    /*Do Something...*/
    grp[1]=1;
    printf("%d",dfs(1));
    return 0;
}

BFS 版本：（其实整体架构和DFS差不多

queue<int>q;		//Queue
int grp[maxn];		//Group
vector<int>g[maxn];	//Graph
bool bfs(int s){
    q.push(s);
    grp[s]=1;
    while(!q.empty()){
        int f=q.front();
        q.pop();
        for(int i=0;i<g[f].size();++i){
            int e=g[f][i];
            if(grp[e]){
                if(grp[e]==grp[f])
                    return 0;
                continue;
            }
            grp[e]=3-grp[f];
            q.push(e);
        }
    }
    return 1;
}
int main(){
    /*Do Something...*/
    printf("%d",bfs(1));
    return 0;
}

例题1.1：劣马

如果用人对应点，人之间的关系对应边……

那不就是二分图判定嘛！

不难想到用map映射人名。

因为关系是相互的，所以要双向存边。

唯一值得注意的地方就是多组数据，一定要好好清空。

#include<map>
#include<queue>
#include<string>
#include<vector>
#include<cstdio>
#include<iostream>
const int maxn=105;
using std::cin;
using std::map;
using std::queue;
using std::string;
using std::vector;
queue<int>q;
string a,b;
int grp[maxn];
int T,m,x,y,tot;
map<string,int>mp;
vector<int>g[maxn];
bool bfs(int s){
    q.push(s);
    grp[s]=1;
    while(!q.empty()){
        int f=q.front();
        q.pop();
        for(int i=0;i<g[f].size();++i){
            int e=g[f][i];
            if(grp[e]){
                if(grp[e]==grp[f])
                    return 0;
                continue;
            }
            grp[e]=3-grp[f];
            q.push(e);
        }
    }
    return 1;
}
inline void init(int c){
    for(int i=1;i<=c;++i){
        g[i].clear();
        grp[i]=0;
    }
    while(!q.empty())
        q.pop();
    mp.clear();
    tot=0;
}
inline void add(int x,int y){
    g[x].push_back(y);
    return;
}
int main(){
    scanf("%d",&T);
    for(int i=1;i<=T;++i){
        scanf("%d",&m);
        while(m--){
            cin>>a>>b;
            if(!(x=mp[a]))
                mp[a]=x=++tot;
            if(!(y=mp[b]))
                mp[b]=y=++tot;
            add(x,y);
            add(y,x);
        }
        printf("Case #%d: %s\n",i,bfs(1)?"Yes":"No");
        init(tot);
    }
    return 0;
}
