layout.ejs 是在页面中使用 layout（默认）进行渲染时用到的文件。

<!DOCTYPE html>
<html>
<head>
    <title></title> <!-- 网站标题 -->
    <meta charset="utf-8"> <!-- 显示中文 -->
</head>
<body>

    这里是 layout.ejs

</body>
</html>

不对啊，这不是 ejs 吗？为什么这些代码是 HTML 代码呢？

因为 ejs 中可以直接写 HTML。真是人性化啊。

然后，重新 hexo s 应该就能看到一行「这里是 layout.ejs」了……

接下来我们学习一下 ejs 的标签语法基础。

Hexo 的渲染模块本身可以使用 Swig 和 ejs，这里我选用了更简单易学、不用安装额外插件的 ejs。

首先，ejs 标签和 html 标签相似，每段代码以「<% + 某字符」开头，以 %> 结尾。而开头的「某字符」，也会决定这段代码的功能。

  • <% %> 表示直接执行里面的 js 代码。

    如：

    <% var Ehe = 'Ehe te nandayo!' %>

    会定义一个名为 Ehe 的变量，其值为 "Ehe te nandayo!"，但不会有输出。

  • <%- %> 表示执行非转义输出。

    这个定义就很迷。我探究了好久才明白这玩意儿是干啥用的。

    就相当于是 cout，你可以 <%- 1 + 1 %>，让页面显示 2，也可以 <%- Ehe %> 或 <%- "Ehe te nandayo!" %>，输出 Ehe te nandayo!。

    注意 <%- %> 会将运算结果 HTML 化，若想避免这一点，可使用 <%= %>，比如：

    <% var str1 = "<di"; %>
<% var str2 = "v> qwq </d"; %>
<% var str3 = "iv>"; %>
<%- str1 + str2 + str3 %>
<%= str1 + str2 + str3 %>

    会输出：

     qwq
<div> qwq </div>

  • <%# %> 是注释。与 /* */ 类似。

    为什么要用 <%# %> 而不是 <!-- --> 呢？答案很简单。当你在 ejs 中进行循环时，循环内的 HTML 会被重复展示，重复的不仅仅是你想要的 <div>，还有注释。许多重复的注释看起来会很傻，所以我们使用单独预处理的 <%# #>。

ejs 本身远远不止标签这一功能。只是因为标签就完全够用了，所以暂时只学标签。

Hexo 预先为我们准备好了一些变量，我们可以在 js 中调用它们。

Hexo 官方网站 中对它们进行了描述，其中，在对变量类型的描述中，objects 类似于结构体变量，array 类似于数组，string 类似于字符串。

*编写完成后 upd： 官方文本 及其模糊 且 及其不准确，尤其是在变量类型方面。在遇到预料外的问题时，建议结合 <% console.log() %> 和 <% typeof() %> 确认准确类型。*

我们发现，Hexo 为我们提供了 <%- partial( PATH OF EJS ) %> 标签，可以将内容放入单独的 ejs 文件，类似于头文件。所以我们可以把刚刚的 layout.ejs 内容更改如下：

<!DOCTYPE html>
<html>
<!-- 页首 -->
<%- partial('_partial/header.ejs') %>
<body>

    <%- body %>
    <!-- 页脚 -->
    <%- partial('_partial/footer.ejs') %>

</body>
</html>

其中 <%- body %> 表示页面的内容，也就是网页渲染 ejs 的内容。

我们可以通过更改网页 index.md 中 Front-matter 的 layout 值，调整其渲染方式。

注意：

  • 所有页面都会使用 layout.ejs 进行渲染，建议在其中写入网站通用的内容，比如侧栏 / 顶栏、页首 / 页脚 的 _partial。
  • 首页（/）的渲染默认为 index.ejs。
  • 归档（/archives，有 s）的渲染默认为 archive.ejs（没有 s），若不存在则为 index.ejs。
  • 分类子页（/categories/*）和标签子页（/tags/*）的默认渲染为 archive.ejs。而分类页（/categories）和标签页（/tags）则需手动 hexo new page 创建。
  • 文章默认为 post.ejs。
  • 其他页面则无默认渲染，需自行填写 layout 项。比如，若想使用 about.ejs 渲染 about 页，则在 source/about/index.md 中修改 layout 为 about。

接下来 hexo s 一下，就可以看到我们在 index.ejs 中的内容了。只是形式较为返璞归真。所以我们就可以愉快地开始写 CSS / js / HTML 了。

还是记录一下一些遇到的坑。

  • 时刻参考 Hexo 的官方文档。其中提供了大量 Hexo 自定义函数、变量的用法，即使它经常是不准确的，但用处也不是一般的大。但注意代码出锅后第一时间 typeof 和 console.log()。

  • ejs 中用 <% %> 括起来的 js 代码，和 <script> 标签中的内容不连通！

    也就是说，你在 <script> 中定义的变量和函数，以及 <%- js() %> 的外部文件内容 ，在 <% %> 是无法使用的。

  • <% %> 中的 console.log() 在 hexo s 时会显示到 bash。

  • 注意新手期常犯的错误，比如路径。

  • 时刻重构代码。当你还不熟练时会写下很多为日后使用带来很多坑的代码。不要犯懒，重构它们。

  • page.ejs 为页面默认渲染。

    如果页面在 Front-matter 中配置了渲染方式，则不使用 page.ejs。也就是说，对于不存在 index.md 文件的主页，会默认使用 page.ejs 和 index.ejs 进行渲染，导致出现问题。解决方式是，在 page.ejs 中使用 Hexo 提供的 is_hone() 函数特判是否为主页。

数学公式设置

参阅：Hexo 主题添加 LaTeX 支持。好险，我上一天看完了这篇文章并完成配置，第二天这个网站就挂掉了。所以写下下面的内容的时候我只能靠记忆自行摸索……

说实话，关于 LaTeX 的配置确实踩了不少坑。之前使用现成的主题，用的是 hexo-renderer-kramed 插件，没想到这次按照同样的方法却不断报错。所以不得不翻遍全网，寻找代替方案。

首先，在主题目录下的 /layout 文件夹中新建 mathjax.ejs，粘贴以下内容：

<% if (theme.mathjax.enable){ %>
    <script type="text/x-mathjax-config">
      MathJax.Hub.Config({
          tex2jax: {
            inlineMath: [ ['$','$'], ["\\(","\\)"] ],
            processEscapes: true
          }
        });
      </script>
    <script type="text/x-mathjax-config">
      MathJax.Hub.Config({
            tex2jax: {
              skipTags: ['script', 'noscript', 'style', 'textarea', 'pre', 'code']
            }
          });
      </script>
    <script type="text/x-mathjax-config">
      MathJax.Hub.Queue(function() {
              var all = MathJax.Hub.getAllJax(), i;
              for(i=0; i < all.length; i += 1) {
                  all[i].SourceElement().parentNode.className += ' has-jax';
              }
          });
      </script>
    <script type="text/javascript" src="<%- theme.mathjax.cdn %>"></script>
<% } %>

然后，在 layout.ejs 的 <body> 中添加：

<% if (page.mathjax == true){ %>
    <%- partial('mathjax.ejs') %>
<% } %>

最后，在 主题的 _config.yml 中添加：

# MathJax Support
mathjax:
  enable: true
  cdn: https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.6/MathJax.js?config=TeX-AMS-MML_HTMLorMML

图标设置

参阅：hexo 的图标配置。

使用纯汉字描述分类和标签等信息会显得臃肿，于是我们引入图标。

首先，引入 Font Awesome 的 CSS。因为其 6.0 版本做出了较大变动，导致与原文内容不符，我们下载其 5.15.4 版本即可。此处是官网下载链接，进入后往下划，点击 Free For Web 即可下载。

解压后，将 /web-fonts-with-css/css 中的所有 CSS 文件拷贝到主题目录的 source/css 中，再将 /web-fonts-with-css/webfont 文件夹拷贝到主题目录的 /sourse 中。

也就是说，此时文件结构如下：

- YourTheme
    - source
        - css
            fa-brands.css
            fa-brands.min.css
            ...
        - webfonts
            fa-brands-400.eot
            fa-brands-400.svg
            ...

在 /layout/_partial/header.ejs 中添加：

<link rel="stylesheet" href="/css/fa-brands.min.css">
<link rel="stylesheet" href="/css/fa-regular.min.css">
<link rel="stylesheet" href="/css/fa-solid.min.css">
<link rel="stylesheet" href="/css/fontawesome.min.css">
<link rel="stylesheet" href="/css/fontawesome-all.css">
<link href="https://use.fontawesome.com/releases/v5.0.4/css/all.css" rel="stylesheet">

接下来，在 这里 搜索你想要的图标，点击图案就可以看到代码啦。在任意你想要显示图标的地方粘贴代码，就可以展示了。

注意无法显示（比如只显示了图标占位符）可能是因为你搜索的不是 Free 图标，重新选中 Free 再搜索试试。
